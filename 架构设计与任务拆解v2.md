## 一、总体架构

采用 **分层模块化** 架构，Flutter 跨平台 UI，核心逻辑用纯 Dart 实现。**文件传输引擎运行在独立 Isolate 中**，与 UI 线程完全隔离，平台差异通过薄通道层抽象。

```
┌──────────────────────────────────────────────────────┐
│  UI Isolate (Flutter Main)                           │
│  ┌──────────────────────────────────────────────────┐│
│  │ Presentation (Widgets)                          ││
│  │ 设备与发现 | 传输 | 历史 | 设置                  ││
│  ├──────────────────────────────────────────────────┤│
│  │ State Management (Riverpod)                     ││
│  └──────────────────────────────────────────────────┘│
├──────────────────────────────────────────────────────┤
│  Transfer Engine Isolate (Dart Isolate)              │
│  ┌──────────────────────────────────────────────────┐│
│  │ TransferCoordinator (消息路由)                  ││
│  │ FileReader / CRC32 / SocketWriter / TokenBucket ││
│  │ TransferModeDecider / DynamicConcurrency        ││
│  └──────────────────────────────────────────────────┘│
├──────────────────────────────────────────────────────┤
│  Business Logic (Pure Dart, runs in UI Isolate)      │
│  ┌──────────┐ ┌───────────┐ ┌──────────────────┐    │
│  │ Discovery│ │ Connection│ │ Clipboard/History│    │
│  │ mDNS/QR  │ │ Manager   │ │ /Settings        │    │
│  └──────────┘ └───────────┘ └──────────────────┘    │
├──────────────────────────────────────────────────────┤
│  Network Transport (Dart:io + custom protocol)       │
│  mDNS Publisher/Scanner | TCP Server/Client         │
├──────────────────────────────────────────────────────┤
│  Platform Abstraction Layer (MethodChannel/FFI)      │
│  Network interfaces | Foreground service |           │
│  Notifications | File access | Drag & Drop | Clipbrd │
├──────────────────────────────────────────────────────┤
│  Host Platform (Android 8+, Windows 10 1809+)        │
└──────────────────────────────────────────────────────┘
```

**关键设计原则**：

- UI 与逻辑完全解耦，业务模块通过 Riverpod 与 UI 通信
- **所有文件 I/O、分片、校验、Socket 写入等 CPU 密集操作在独立 Engine Isolate 执行**，UI 线程仅接收进度事件
- 网络传输层提供统一接口，未来可替换为 QUIC 等协议
- 平台通道仅暴露最小必要功能，业务不依赖平台

## 二、模块职责

### 2.1 设备发现模块 (DiscoveryService)

- 基于 mDNS：发布服务 `_fastshare._tcp.local.`，持续扫描
- 维护在线设备列表（去重、心跳、离线移除）
- 多网卡绑定：根据用户选择或自动策略绑定特定接口
- 输出设备上线/下线事件流

### 2.2 连接与配对管理 (ConnectionManager)

- 信任设备存储（SQLite / JSON）
- TCP 长连接池：按需建立，空闲回收，支持重连
- 配对流程：6 位随机码挑战，确认后永久信任
- 连接认证：信任设备免确认，非信任走确认或白名单自动接受

### 2.3 文件传输引擎 (TransferEngine)

**完全运行在独立 Dart Isolate 中**，通过 `SendPort`/`ReceivePort` 与主 UI Isolate 通信。

- **队列管理**：任务排队、暂停/恢复、取消
- **传输模式判定器**：根据待传文件自动选择顺序/并发/混合模式
- **动态并发调整器**：监测吞吐量、IO、内存、帧率，自动微调
- **块传输控制器**：分块、ACK、断点续传
- **文件夹处理**：序列化目录结构，先建树后传文件
- **速率控制**：令牌桶算法，支持手动限速
- **聚合发送（可选）**：极小文件打包
- **进度汇报**：经去抖处理，以不超过 10 次/秒的频率向 UI Isolate 发送进度事件
- **文件列表扫描**：在 Engine Isolate 中异步分批遍历，每 50 个文件向 UI 推送中间结果
- **资源释放**：传输完成或取消后 Engine Isolate 自动销毁，避免空转

### 2.4 剪贴板共享模块

- 通过现有 TCP 连接推送文本，接收端写入系统剪贴板

### 2.5 历史记录与设置

- 历史记录：SQLite 存储，可查询、清理
- 用户设置：信任列表、并发、限速、网卡、深色模式、保护阈值等（SharedPreferences + sqflite）

### 2.6 状态管理 (Riverpod)

主要 Provider：

- `discoveryProvider`：在线设备流
- `knownDevicesProvider`：已知信任设备
- `transferQueueProvider`：传输任务状态机
- `activeTransfersProvider`：活跃传输实时进度（来自 Engine Isolate 事件）
- `settingsProvider`：用户配置
- `networkInterfacesProvider`：可用网卡列表

### 2.7 平台通道 (PlatformChannel)

抽象接口 `PlatformInterface`，各平台实现：

- `getNetworkInterfaces()` → IP、网卡名、类型
- `startForegroundService(config)` → 通知栏/系统托盘
- `showNotification(id, title, body, progress)` → 进度通知
- `getSharedFiles()` → Android 接收系统分享
- `registerDropTarget(windowId)` → Windows 拖拽
- `copyToClipboard(text)` / `getClipboardText()`
- `openFileLocation(path)` → 打开系统文件浏览器
- `getBatteryLevel()` / `getThermalState()` → 保护信息
- `getStorageInfo(path)` → 磁盘剩余空间

## 三、Transfer Engine Isolate 详细设计

### 3.1 设计动机

Dart 单线程事件循环中，文件分片、CRC32 校验、大量磁盘 I/O 等操作即使使用异步 Future，仍会抢占 CPU 时间导致 UI 卡顿。必须将传输引擎独立到一个专用 Isolate，利用多核并保证 UI 线程绝对流畅。

### 3.2 消息通信模型

```
UI Isolate                          Engine Isolate
   │                                    │
   │ ──── start_transfer ────────────►  │
   │                                    ├─ 扫描文件列表
   │ ◄─── file_list_chunk ────────────  │
   │                                    ├─ 判定模式，开始传输
   │ ◄─── progress (10/s) ────────────  │
   │ ◄─── speed ─────────────────────── │
   │ ◄─── file_complete ──────────────  │
   │ ◄─── error ──────────────────────  │
   │                                    │
   │ ──── pause / resume / cancel ───►  │
   │                                    │
   │ ◄─── transfer_complete ──────────  │
   │                                    ├─ 销毁 Isolate
```

### 3.3 命令与事件定义

```dart
// UI → Engine
class EngineCommand {
  String type; // start_transfer, pause, resume, cancel, shutdown
  Map<String, dynamic> payload;
}

// Engine → UI
class EngineEvent {
  String type; // progress, speed, file_complete, error, mode_change, file_list_chunk
  Map<String, dynamic> data;
}
```

### 3.4 进度事件去抖

Engine Isolate 内部每 1 秒采样一次性能数据，但向 UI 发送的进度更新限制为 **每 100ms 最多一次**（即 10 次/秒），避免过度刷新。

```dart
Timer? _progressTimer;
void _onProgress() {
  if (_progressTimer != null) return;
  _progressTimer = Timer(const Duration(milliseconds: 100), () {
    _progressTimer = null;
    _sendToUI(EngineEvent('progress', _currentState));
  });
}
```

### 3.5 文件列表扫描流程

1. UI Isolate 发送 `start_transfer` 命令，携带根目录路径
2. Engine Isolate 使用 `Directory.list(recursive: true)` 流式遍历
3. 每收集 50 个文件，发送 `file_list_chunk` 事件给 UI（UI 可渐进式渲染）
4. 若 UI 发送 `cancel` 或根目录极深，Engine 可提前结束扫描，使用已收集文件开始传输
5. 扫描完成或开始传输后停止发送列表事件

### 3.6 特别注意事项

- **Isolate 生命周期**：传输完成/取消后立即 `kill()` 释放资源；无传输任务时不保留空转 Isolate
- **Socket 转移**：连接建立由 UI Isolate（或轻量网络 Isolate）完成，随后通过 `IsolateNameServer` 或直接转移 `Socket` 至 Engine Isolate 进行数据传输
- **文件权限**：Android content URI 等需在主 Isolate 获取读权限，将路径传给 Engine，Engine 只负责读取文件内容
- **错误隔离**：Engine Isolate 内部未捕获异常不会导致 UI 崩溃，会封装为 `error` 事件发送给 UI，UI 提示“传输引擎异常，请重试”

## 五、通信协议概要（遵循 FLP v1.2）

- **服务发现**：mDNS `_fastshare._tcp`，TXT 携带设备 ID、名称、头像哈希、版本
- **传输端口**：TCP 默认 45678（可配置）
- **消息格式**：
  - 控制消息：UTF-8 JSON/CBOR 行，封装在统一 Frame 中
  - 数据块：二进制结构体，包含 chunk 索引、偏移、数据及校验
- **关键特性**：版本协商、滑动窗口 ACK、断点续传、小文件聚合、剪贴板推送
- **强制约束**：UI Isolate 不得直接接触 Socket 及协议解析，所有 FLP 操作必须在 Engine Isolate 完成
